# Connection Audit

## Connection model

- Transport: REST over local-network HTTP
- Discovery: QR payload generated by the Pi dashboard
- Auth: Bearer token issued by `POST /pair`
- Sync mode: request/response polling via `POST /sync`

Not implemented:

- Bluetooth
- hotspot orchestration
- mDNS/service discovery
- cloud relay
- WebSocket sync

## Confirmed contract alignment

### Matching endpoints

Android calls the same routes the Pi exposes:

- `GET /health`
- `GET /time`
- `GET /templates`
- `POST /pair`
- `POST /sync`

### Matching field names

- Both sides use snake_case JSON
- Android DTOs match Pi model names for:
  - `bag_id`
  - `template_id`
  - `updated_at`
  - `updated_by`
  - `packed_status`
  - `expiry_date_ms`
  - `last_sync_at`
  - `server_item_changes`
  - `server_bag_changes`
  - `server_version`

### Matching port assumptions

- Code and docs consistently target port `8080`
- The earlier `8000` mismatch described in `docs/llm-project-context.md` is not present in the current pairing screen code

## Confirmed flow

1. Pi dashboard renders a QR payload containing:
   - `base_url`
   - `pair_code`
   - `pi_device_id`
2. Android scans that payload.
3. Android calls `POST /pair` with:
   - `phone_device_id`
   - `pair_code`
4. Pi returns:
   - `auth_token`
   - `pi_device_id`
   - `server_time_ms`
5. Android stores credentials and then pulls templates.
6. Android later calls `POST /sync` with changed bags/items since `last_sync_at`.

## Confirmed weak points

- Pairing depends entirely on the QR's `base_url` being reachable from the phone.
- The Pi derives `base_url` from the request `Host` header unless `GOBAG_BASE_URL` is set. On real networks, that can produce bad QR payloads if the dashboard is opened through the wrong hostname or port.
- Sync requires a bearer token, but `GET /templates` does not.
- There is no automatic retry/backoff strategy beyond Android exception handling.

## Mismatches found

No critical schema mismatch was found between the Android Retrofit DTOs and the FastAPI models.

The main connection-level gaps are operational rather than structural:

- no automatic discovery
- no TLS
- no authenticated admin routes
- no network setup assistance

## Status

- Code coherence: Good
- First-pair usability on a healthy LAN: Likely good
- Connection robustness in messy home-network conditions: Moderate risk
- Zero-config connectivity: Not achieved
