# GO BAG Configuration

This document covers the install and deployment values that affect the Raspberry Pi backend and the Android pairing/sync flow.

## Canonical Pi config file

The installed Raspberry Pi config file is:

- `/opt/gobag/config/gobag.env`

The example templates kept in git are:

- `pi-server/config/gobag.env.example`
- `pi-server/.env.example`

Keep the example files in the repository. Do not commit a real populated `gobag.env`.

## Raspberry Pi environment variables

### `GOBAG_HOST`

- default: `0.0.0.0`
- used by the backend process binding
- leave this as `0.0.0.0` unless you intentionally want to restrict access to a specific interface

### `GOBAG_PORT`

- default: `8080`
- used by the backend process and all local verification commands
- if you change it, update any Android address you enter manually and any firewall/router rules

### `GOBAG_BASE_URL`

- default: blank
- when blank, the backend tries to detect a reachable non-loopback IP and builds a base URL automatically
- affects the URLs used for the dashboard, QR payload, and advertised pairing address

Set this manually when:

- the detected address is wrong
- the Pi is on a hotspot and you want a fixed hotspot URL
- the Pi has multiple interfaces
- the QR code should always advertise a specific address

Example:

```env
GOBAG_BASE_URL=http://192.168.1.20:8080
```

Override behavior:

- non-empty `GOBAG_BASE_URL` wins
- otherwise the backend uses a detected non-loopback IP
- if detection fails, it falls back to `http://127.0.0.1:<port>`

### `GOBAG_DATA_DIR`

- installed default: `/opt/gobag/data`
- backend database path is `${GOBAG_DATA_DIR}/gobag.db`
- if changed, keep the directory writable by the user running `gobag-backend.service`

### `GOBAG_LOG_DIR`

- installed default: `/opt/gobag/logs`
- used by launcher scripts for `launcher.log` and `launcher-backend.log`

### `GOBAG_BACKUP_DIR`

- installed default: `/opt/gobag/backups`
- created by the installer for Pi-side data management

### `GOBAG_DEVICE_NAME`

- default: `GO BAG Raspberry Pi`
- shown in device status responses and pairing feedback

### `GOBAG_ADMIN_TOKEN`

- default in example file: blank
- generated by `pi-server/install.sh` if missing
- used for admin actions on the Pi side
- never commit a real generated admin token

### `GOBAG_AUTO_OPEN_UI`

- default: `1`
- used by the launcher flow to decide whether the native GO BAG app shell opens automatically

### `GOBAG_UI_SHELL`

- default: `pywebview`
- selects the local Raspberry Pi UI shell implementation

### `GOBAG_APP_URL`

- default: blank
- when blank, the local app shell uses `http://127.0.0.1:<port>` so the touchscreen UI works without internet access

### `GOBAG_APP_FULLSCREEN`

- default: `1`
- starts the local app shell in fullscreen mode for the 3.5-inch touchscreen

### `GOBAG_APP_FRAMELESS`

- default: `1`
- removes normal window chrome from the local app shell

### `GOBAG_APP_WIDTH`

- default: `480`
- fallback window width for non-fullscreen launches

### `GOBAG_APP_HEIGHT`

- default: `320`
- fallback window height for non-fullscreen launches

### `GOBAG_APP_WAIT_TIMEOUT_S`

- default: `25`
- how long the app shell waits for the local backend to become healthy before failing

### `GOBAG_BROWSER_CMD`

- default: blank
- optional fallback browser command for troubleshooting only

### Camera-related values

These exist in the current backend and example config:

- `GOBAG_ENABLE_CAMERA`
- `GOBAG_CAMERA_CMD`
- `GOBAG_CAMERA_WIDTH`
- `GOBAG_CAMERA_HEIGHT`
- `GOBAG_CAMERA_WARMUP_MS`
- `GOBAG_CAMERA_TIMEOUT_S`

They affect camera status and capture behavior, but they are not required for Android pairing and sync to work.

## Example Pi config

```env
GOBAG_HOST=0.0.0.0
GOBAG_PORT=8080
GOBAG_DEVICE_NAME="GO BAG Raspberry Pi"
GOBAG_DATA_DIR=/opt/gobag/data
GOBAG_LOG_DIR=/opt/gobag/logs
GOBAG_BACKUP_DIR=/opt/gobag/backups
GOBAG_BASE_URL=http://192.168.1.20:8080
GOBAG_ADMIN_TOKEN=
GOBAG_AUTO_OPEN_UI=1
GOBAG_UI_SHELL=pywebview
GOBAG_APP_URL=
GOBAG_APP_TITLE="GO BAG Inventory"
GOBAG_APP_FULLSCREEN=1
GOBAG_APP_FRAMELESS=1
GOBAG_APP_WIDTH=480
GOBAG_APP_HEIGHT=320
GOBAG_APP_WAIT_TIMEOUT_S=25
GOBAG_BROWSER_CMD=
GOBAG_ENABLE_CAMERA=1
GOBAG_CAMERA_CMD=libcamera-still
GOBAG_CAMERA_WIDTH=1280
GOBAG_CAMERA_HEIGHT=720
GOBAG_CAMERA_WARMUP_MS=900
GOBAG_CAMERA_TIMEOUT_S=12
```

## Android endpoint and pairing state

The Android app does not use a checked-in `.env` file. Instead, it stores pairing and endpoint state locally in DataStore.

Current stored values include:

- `base_url`
- `auth_token`
- `pi_device_id`
- `last_sync_at`
- `connection_status`
- `pending_changes_count`
- `local_ip`
- `last_connection_error`
- `selected_bag_id`

From the current code:

- saving an address stores `base_url` only
- QR pairing stores `auth_token`, `base_url`, and `pi_device_id`
- first sync updates `last_sync_at` and status metadata

Important practical rule:

- an entered or saved endpoint is not the same thing as pairing

## Android local persistence

The Android app currently stores:

- Room database name: `gobag.db`
- DataStore file name: `device_state.preferences_pb`

These are device-local runtime files and should never be committed to git.

## When `GOBAG_BASE_URL` must usually be set manually

You will usually want to set `GOBAG_BASE_URL` yourself when:

- Android keeps reaching the wrong Pi address
- the Pi’s dashboard QR code advertises an unusable address
- your network changes often and auto-detection chooses the wrong interface
- you are presenting or deploying in a hotspot-only setup

After changing `/opt/gobag/config/gobag.env`, restart the service:

```bash
sudo systemctl restart gobag-backend.service
```

Then verify:

```bash
curl http://127.0.0.1:8080/device/status
```

Check that `local_ip` and the dashboard/QR behavior now match the intended address.

## Safe example-file handling

Recommended pattern:

- keep only example files in git
- copy example files when setting up a machine
- fill in real local values only on the machine
- keep real tokens, database paths, and machine-local values out of version control
